# 자주 쓰는 프롬프트

## 세션 시작
"내 Obsidian 볼트의 프로젝트/[프로젝트명]/ 폴더를 전부 읽고,
현재 프로젝트 상태를 요약해줘."

## 설계 요청 (지휘자용)
"[기능]을 추가하려고 해. 
아키텍처.md를 읽고, 기존 구조에 어떻게 통합하면 좋을지 설계해줘.
데이터 구조 변경이 필요하면 마이그레이션 계획도 포함해줘."

## 구현 지시 (현장 책임자용)
"아키텍처.md와 [Phase]-지시서.md를 읽고,
해당 Phase를 구현해줘.
각 단계 끝날 때마다 동작 확인해줘."

## 세션 종료
"지금까지 한 일을 요약해줘.
어떤 파일을 만들었고, 어떤 기능이 완성됐고,
다음에 뭘 해야 하는지."

## 에러 에스컬레이션 (지휘자용)
"[기능] 구현 중에 에러가 반복되고 있어.
증상: [증상]
시도한 것: [시도 내용]
아키텍처.md를 보고 설계 수준에서 원인이 있는지 분석해줘."
